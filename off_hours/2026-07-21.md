## Methodology fix: MACDh was actually the signal line
`replay_check.py` read `macd_df.iloc[:, 2]` and called it `macd_hist`. pandas_ta's real column
order is `[MACD, MACDh, MACDs]` — index 2 is the **signal line**, not the histogram (index 1).
Every "MACDh flip" exit test run before today (including yesterday's off-hours check below) was
actually testing signal-line sign flips, a related but different signal. Fixed — MACD line,
histogram, and signal line are now stored and used correctly.

Also extended the harness with a real v1.2 variant (previously only v1.0 vs v1.1 existed), and
switched the ticker list from the stale 9-ticker default (set when the script was written Jul 18)
to the actual current 22-ticker watchlist+holdings universe.

than isolating just the new risk rules.

**Recommendation for tonight's evolve**: the empirical case for both v1.1 and v1.2 as improvements
over v1.0 looks weak on this corrected test. Worth treating v1.2's promotion as unsettled rather
than validated, and considering whether triple-confirmation is entering too often/too loosely
(170 trades over 200d/22 tickers is a lot of turnover for a "wide and diverse, high-conviction"
mandate).
